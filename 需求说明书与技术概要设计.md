# DHL 深圳枢纽产能预测系统 — 需求说明书与技术概要设计

## 一、核心使用流程

```
初始化：导入3-5年历史数据（Excel）
         ↓
    配置航线 & 运力基准
         ↓
┌─→ AI 生成预测（未来1天/7天/30天）──→ 展示预测走势图
│        ↓
│   次日：用户提交昨日实际数据
│        ↓
│   系统自动回溯偏差归因（AI分析）
│        ↓
│   实际数据追加入历史库
│        ↓
└───── 模型基于更新后的数据重新预测
```

**关键闭环：预测 → 实际 → 偏差归因 → 数据沉淀 → 优化下一次预测**

## 二、功能需求

### F1. 数据导入与管理

- 支持 Excel 批量导入历史货量数据（按航线/日期）
- 支持每日追加昨日实际数据（货量、装载率等）
- 数据校验：缺失值提示、异常值标记
- 数据列表可查看、可筛选、可导出

### F2. 航线与运力基准配置

- 维护航线信息（US/EU/iAN 等）
- 每条航线：航班排期、机型、单班运力
- 按季度配置，系统自动算出周/月运力上限
- 运力基准作为预测走势图中的固定参考线

### F3. 日历与事件标注

- 内置公历节假日（中国+主要目的地国）
- 支持手动添加事件（大促、政策变动、展会等）
- 每个事件可配置：影响航线、预估影响幅度（%）、持续时间
- 事件在走势图上以标记点形式显示

### F4. 外部情报抓取

系统内置一批推荐数据源，每次预测前自动抓取最新信息，作为 AI 预测的输入上下文。用户可在后台增删数据源。

**预置推荐数据源：**

| 类别 | 数据源 | URL | 抓取内容 |
|------|--------|-----|---------|
| 宏观经济 | 国家统计局 | stats.gov.cn | PMI 制造业指数 |
| 海运运价 | 上海航运交易所 | sse.net.cn | SCFI 综合指数 |
| 汇率 | 中国人民银行 | pbc.gov.cn | 人民币中间价 |
| 天气 | 中央气象台 | nmc.cn | 深圳及航线目的地天气预警 |
| 航空通告 | 民航局 | caac.gov.cn | NOTAM / 空域管制通知 |
| 贸易政策 | 海关总署 | customs.gov.cn | 关税调整、贸易政策公告 |
| 电商动态 | 各平台公告 | — | 大促日期、招商节奏 |

**工作机制：**
- 抓取方式：后端定时任务 或 预测前手动触发
- 抓取结果存入 `external_signals` 表（日期、来源、类别、原文摘要、数值）
- AI 预测时自动拉取近期外部信号作为上下文
- 用户可在"数据源管理"页面：查看抓取历史、启用/禁用数据源、添加自定义数据源（填入 URL + 抓取规则描述）

### F5. AI 预测引擎

**输入给 AI 的上下文：**
- 近期历史数据（近30-90天实际货量）
- 航线运力基准
- 未来已知事件（节假日、大促等）
- 历史偏差归因记录（作为经验知识）
- **最新外部情报**（PMI、运价、汇率、天气、政策等）

**AI 输出：**
- 未来1天预测值（按航线）
- 未来7天每日预测值（按航线）
- 未来30天每日预测值（按航线）
- 每个预测附带置信区间（高/中/低三档）
- 关键影响因素说明（文字）

**触发时机：**
- 用户提交每日实际数据后，自动触发重新预测
- 用户也可手动触发重新预测

### F6. 预测走势图（核心页面）

一张图上叠加以下元素：
- **运力上限线**：水平虚线（来自航线基准配置）
- **历史实际货量**：实线（已发生的真实数据）
- **AI 预测货量**：虚线 + 置信区间带（未来部分）
- **最优装载区间**（85%-95%）：浅绿色背景带
- **事件标记**：图上对应日期标注事件名称
- 时间范围切换：近7天 / 近30天 / 近90天
- 航线筛选：可选单条航线或全部汇总

### F7. 每日数据提交与偏差归因

**提交流程：**
1. 用户进入"每日数据提交"页面
2. 填写昨日各航线实际货量（或 Excel 导入）
3. 提交后，系统自动：
   - 将数据写入历史库
   - 计算昨日预测值 vs 实际值的偏差
   - 调用 AI 分析偏差原因

**偏差归因展示：**
- 偏差率（%）+ 绝对值
- AI 生成的归因分析（文字说明）
- 归因标签（系统建议 + 用户可修改）：
  宏观经济 / 政策 / 季节性 / 电商 / 竞争 / 海运替代 / 天气 / 突发事件
- 用户确认或修正归因后，存入归因知识库

### F8. 归因知识库

- 所有历史偏差归因的列表视图
- 按标签分类统计（饼图）
- 按时间趋势展示（堆叠图）
- 同类归因 ≥3 次时高亮提示"已形成模式"
- 知识库数据会作为上下文传给 AI，提升预测精度

### F9. 装载率仪表盘

- 各航线当前/预测装载率（仪表盘样式）
- 红黄绿三色状态
- 点击可展开该航线详细趋势

### F10. 决策建议

- 基于装载率状态，AI 生成运营建议
- 运力过剩：降价建议、合班建议
- 运力不足：提价建议、加班机成本估算
- 建议以卡片形式展示，可标记"已采纳/忽略"

### F11. What-if 模拟

- 用户输入假设条件（如：增加一班美线航班 / 黑五提前一周）
- AI 基于当前数据 + 假设条件重新预测
- 对比展示：原预测 vs 模拟预测

### F12. 预测回测

- 过去已产生预测的日期，展示"当时预测 vs 实际结果"
- 计算历史预测准确率（MAPE）
- 趋势图展示预测精度随时间的变化

## 三、技术概要设计

### 3.1 技术栈

| 层 | 选型 | 理由 |
|----|------|------|
| 前端 | Next.js (React) + TailwindCSS | 全栈框架，SSR + API 一体 |
| 图表 | ECharts | 走势图/仪表盘/热力图，生态成熟 |
| 数据库 | SQLite (better-sqlite3) | 本地部署，无需额外服务 |
| AI 接口 | GLM-4.7（智谱） | 预测 + 归因 + 建议生成 |
| 网页抓取 | cheerio + fetch | 轻量抓取官方网站信息 |
| 文件处理 | xlsx (SheetJS) | Excel 导入导出 |

### 3.2 数据库设计（核心表）

```
routes           航线表（航线代码、名称）
fleet_config     运力配置表（航线、机型、航班排期、单班运力、生效季度）
cargo_history    历史货量表（日期、航线、实际货量、装载率）
events           事件表（名称、类型、影响航线、影响幅度、起止日期）
external_sources 外部数据源表（名称、URL、类别、抓取规则、启用状态）
external_signals 外部情报表（日期、数据源ID、类别、摘要、数值）
predictions      预测记录表（预测日期、目标日期、航线、预测值、置信区间）
deviations       偏差归因表（日期、航线、预测值、实际值、偏差率、归因标签、AI分析、用户修正）
decisions        决策建议表（日期、航线、建议内容、状态）
simulations      模拟记录表（假设条件、原预测、模拟预测）
```

### 3.3 AI 调用设计

**模型配置：**
- 模型：GLM-4.7（智谱清言）
- 接口：`POST https://open.bigmodel.cn/api/coding/paas/v4/chat/completions`
- API Key：`59c4af77e1534f278f49fd3f56159eab.auO89JqQFYEZnVbV`

所有 AI 调用统一封装为 prompt 工程，不依赖微调：

| 场景 | 输入 | 输出 |
|------|------|------|
| 情报摘要 | 抓取的网页原文 | 结构化摘要 + 关键数值提取 |
| 预测 | 历史数据 + 基准 + 事件 + 归因知识 + 外部情报 | 未来1/7/30天预测值 + 置信区间 + 因素说明 |
| 偏差归因 | 昨日预测vs实际 + 近期数据 + 事件 | 偏差原因分析 + 建议标签 |
| 决策建议 | 装载率 + 偏差趋势 + 历史弹性 | 定价/运力调配建议 |
| What-if | 当前数据 + 用户假设条件 | 模拟预测结果 |

### 3.4 页面结构

```

/data/history         历史数据管理

/config/routes        航线与运力配置
/config/events        事件管理
/config/sources       外部数据源管理
/analysis/attribution 归因知识库

/simulation           What-if 模拟

```
