# DHL 深圳枢纽产能预测系统 — 整体框架

## 一、项目目标

为枢纽协调管理员提供 **未来7-14天** 的产能需求预测，核心解决：

- 每季度航班固定 → 运力上限已知
- 实际货量波动 → 与运力产生差值
- 差值归因沉淀 → 反哺预测精度
- 驱动市场定价 + 运力调配决策

---

## 二、核心决策链路

```
季度航班计划(固定) → 总运力上限(已知)
                          ↕ 对比
               实际/预测货量(变动)
                          ↓
                     差值分析
                    ↙        ↘
            运力过剩            运力不足
               ↓                  ↓
         归因 → 沉淀            归因 → 沉淀
               ↓                  ↓
         降价促量/减班合并       提价/加班机/转港
```

## 三、数据输入层

## 四、运力基准线（季度固定）

每季度初确定，作为系统的"天花板"：

| 维度 | 示例 |
|------|------|
| 航线 | US / EU / iAN / 其他 |
| 每条航线周航班数 | US线：周一/三/五/日 共4班 |
| 单班运力 | B777F → 约100吨 / ~12,000件 |
| **周运力上限** | 航班数 × 单班运力 |
| **季度总运力** | 周运力 × 13周 |

## 五、预测算法

### 5.1 基准货量预测

```
预测货量 = 近期趋势基数 × 同期季节性系数

近期趋势基数 = 最近4周加权均值（权重：0.4/0.3/0.2/0.1）
同期季节性系数 = avg(往年同期货量) / avg(往年同期前4周货量)
```

**同期数据对齐规则：**
- 按春节相对周坐标对齐，而非日历日
- 排除异常年份（如2020疫情），取3-5年加权均值
- 近年权重更高（例：2024×0.4, 2023×0.3, 2022×0.2, 2019×0.1）

### 5.2 事件修正

```
修正后货量 = 基准货量 × (1 + Σ修正因子)

修正因子包括：
- 政策因子：关税调整 → 抢出口效应（+10%~30%）
- 促销因子：黑五前2周 → 出口激增（+15%~25%）
- 天气因子：台风预警 → 滞留堆积后延后释放
- 海运替代因子：海运运价涨超阈值 → 空运+5%~15%
- 归因知识库因子：历史沉淀的同类事件影响系数
```

### 5.3 货量 → 产能需求换算

```
ULD需求 = 预测货量(件) × 平均件重 / ULD平均装载量
         按航线拆分（US / EU / iAN）

人力需求 = 预测货量(件) / 目标PPH × 冗余系数(1.15)
         按班次拆分（早/中/晚）

设备需求 = f(ULD数量, 航班密度, 仓库吞吐峰值)
```

## 六、差值分析 & 归因沉淀

### 6.1 差值计算

```
差值 = 实际货量 - 运力上限
装载率 = 实际货量 / 运力上限 × 100%

装载率 85-95%  →  最优状态（绿色）
装载率 < 85%   →  运力过剩（黄色）
装载率 > 95%   →  运力不足（红色）
```

### 6.2 归因标签体系

| 类别 | 标签 | 典型影响 |
|------|------|---------|
| 宏观经济 | PMI下行 / 汇率波动 | ±5-15% |
| 政策 | 关税调整 / 贸易限制 | ±10-30% |
| 季节性 | 春节淡季 / 旺季 | ±15-25% |
| 电商 | 黑五 / 双11 / Prime Day | +15-30% |
| 竞争 | 客户流失 / 竞对降价 | -5-15% |
| 海运替代 | 海运涨价转空运 | +5-15% |
| 天气 | 台风 / 暴雨 | 延后释放 |
| 突发事件 | 红海危机 / 罢工 | ±10-20% |

### 6.3 沉淀规则

- 差值超过 **±10%** 时，系统强制管理员选择归因标签
- 归因方式：下拉选择标准标签 + 自由文本补充
- 同类归因出现 **3次以上** → 自动纳入预测模型修正因子
- 每季度回顾归因知识库，清理过时条目

## 七、决策输出

### 7.1 市场端（定价 & 销售）

**运力过剩时：**
- 降价幅度建议（基于历史弹性系数：降价10%通常带来+8%货量）
- 定向促销目标客户/品类
- 预计多久能填满（结合Booking趋势）

**运力不足时：**
- 提价幅度建议（基于历史：涨价5%流失约3%货量）
- 低价值货物 Deflect 建议（转下一班/其他枢纽）
- 高价值客户优先保障排序

### 7.2 运营端（运力调配）

**运力过剩时：**
- 合并航班 / 缩减舱位
- 释放ULD给其他枢纽
- 减少临时工班次

**运力不足时：**
- 加班机成本收益计算
- 转港方案（SZX溢出 → HKG消化）
- 增加班次 / 临时工需求

## 八、可视化看板

### 8.1 运力走廊图（核心视图）

横轴为时间（周），纵轴为货量：
- 运力上限：固定水平线
- 实际/预测货量：波动曲线
- 最优区间（85-95%）：绿色带
- 溢出区 / 空舱区：红/黄色标注
- 点击任意一周 → 弹出差值归因 + 历史同期对比

### 8.2 装载率仪表盘

按航线（US/EU/iAN）展示当前周装载率，颜色编码一目了然。

### 8.3 ULD需求热力图

按日期×航线矩阵，颜色标注需求紧张程度：
- 红色 = 需求超当前库存
- 黄色 = 接近上限
- 绿色 = 充足

### 8.4 人力排班建议

各班次当前排班 vs 建议排班，标注增减人数。

### 8.5 风险预警

- 未来7天的关键风险事件
- 预计影响幅度
- 建议应对方案

### 8.6 归因趋势分析

过去12个月运力偏差的归因分布（饼图/堆叠图），识别系统性问题。

### 8.7 预测回测

展示模型预测 vs 实际结果的偏差，建立管理员对系统的信任。

## 九、管理员交互功能

- **拖拽修正**：手动调整预测曲线，系统记录修正原因，反哺模型
- **What-if 模拟**：输入假设条件（如"增加一个美线航班"），自动重算ULD和人力需求
- **一键排班**：基于预测结果生成ULD调拨单 + 人力排班表，确认后下发

## 十、实施路径

| 阶段 | 内容 | 周期 | 产出 |
|------|------|------|------|
| Phase 1 | 历史数据清洗 + 季节性模型 + 运力走廊图 + 手动归因 | 4周 | 可用的基础看板，预测准确率80-85% |
| Phase 2 | 接入Booking/航班实时数据 + 事件修正因子 + 自动排班建议 | 4周 | 实时预测 + 运营建议 |
| Phase 3 | ML模型（XGBoost/Prophet）+ 归因知识库自动反哺 + What-if模拟 | 6周 | 预测准确率90%+ + 完整决策支持 |

## 十一、价值闭环

```
预测货量 → 对比固定运力 → 发现差值 → 归因沉淀
    ↑                                    ↓
    ←──── 反哺预测模型 ←──── 知识库积累
                                         ↓
                                驱动市场定价 + 运力调配
```

Phase 1 即可产出可用成果，管理员用起来后持续沉淀归因数据，为后续 ML 模型提供高质量训练样本。
